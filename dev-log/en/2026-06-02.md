# 2026-06-02 Development Log

Today I mainly started reading `useWorldEcho.ts`, with the goal of understanding the real Echo loop more clearly.

## `useWorldEcho.ts`

This file now owns the Echo-specific flow:

```text
player opens note paper
  -> writes a Note Echo
  -> sends it to LLM / fakeResolver
  -> receives the agent reaction
  -> computes state drift and relationship drift
  -> updates world state
```

### Core data flow

```text
player writes Note Echo
  -> sendNoteEcho()
  -> buildNoteEchoInput()
  -> assemble AgentBrainInput
  -> resolveAgentBrainLlm()
  -> call real LLM; fallback to fakeResolver on failure
  -> resolveEchoOutcome()
  -> compute stateDrift + relationshipDrift + DailyEchoRecord
  -> dispatch({ type: "echo/applied", ... })
  -> worldReducer updates world state
```

### `sendNoteEcho()`

`sendNoteEcho()` is the core function in this file.

It first reads the player's current draft:

```ts
state.note.draft
```

Then it uses `.trim()` to remove leading and trailing whitespace.

If the player did not write anything:

```ts
if (!noteText) return;
```

then nothing is sent.

### `echoApiUrl`

```ts
const echoApiUrl = import.meta.env.VITE_ECHO_API_URL;
```

This is the Vite way of reading an environment variable. It means: read `VITE_ECHO_API_URL` from `.env` and use it as the target URL for the `/echo` request.

### Real LLM call with fake fallback

```ts
void resolveAgentBrainLlm({
  apiUrl: echoApiUrl,
  input,
  fallbackResolver: resolveAgentBrainFake,
}).then((brainOutput) => {
  ...
});
```

The meaning here is:

- try the real LLM resolver first
- if it fails, fall back to `resolveAgentBrainFake`

The `void` here does not mean “no result.” It means:

- I know this returns a Promise
- but I do not `await` it here
- and I do not return that Promise to the caller

### `resetForNextDay()`

```ts
const resetForNextDay = () => {
  dayStartAgentState.current = latestAgentState.current;
  dayStartRelationships.current = latestRelationships.current;
  latestDailyEchoes.current = [];
};
```

This function is used when the game enters the next day.

It does three things:

1. Use the end-of-day `agentState` as the next day's starting state
2. Use the end-of-day `relationships` as the next day's starting relationships
3. Clear `dailyEchoes`

## The most important understanding in this layer

The key point of `useWorldEcho.ts` is still `sendNoteEcho()`:

```text
player writes one sentence
  -> get noteText
  -> assemble AgentBrainInput
  -> send to LLM
  -> LLM returns brainOutput
  -> resolveEchoOutcome performs local rule resolution
  -> update agentState / relationships / dailyEchoes
  -> dispatch into worldReducer
```

So this layer is not simply “send one sentence to the model.” It is the place where the player's signal is connected back into local world rules.

## One TypeScript note

`AgentBrainInput["event"]` is a TypeScript **indexed access type**.

It means:

- start from the `AgentBrainInput` type
- extract the type of its `event` field

It reads a **type**, not a runtime value.
