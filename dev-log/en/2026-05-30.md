### `agentReactionSchema`

`agentReactionSchema` defines what reactions the agent can have.

When the player sends an Echo, the agent does not have to accept it directly. She can respond in 6 ways:

| reaction | meaning |
|---|---|
| `accepted` | received it |
| `hesitated` | hesitated |
| `resisted` | resisted it |
| `misread` | misread it |
| `delayed` | responded later |
| `transformed` | turned it into another understanding |

Core code:

```ts
const response = await fetch(apiUrl, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    chart: buildAstrologyChartSeed(input),
  }),
});
```

This means: send a `POST` request to `apiUrl`, and send the `chart` data to the backend as JSON.

`JSON.stringify` means: turn an object into a JSON string. It is not just wrapping the object in `""`; it serializes the whole JS object into JSON format.

`Content-Type` means: tell the backend, "I am sending JSON."

```ts
"Content-Type": "application/json";
```

So the data type of the request `body` is `application/json`. The server should not treat this `body` as plain text, a form, or an image. It should parse it as JSON.

The second parameter:

```ts
{
  method: "POST",
  headers: { ... },
  body: ...
}
```

is the request configuration object. So:

```ts
fetch(apiUrl, { ... })
```

means: call `fetch` to request `apiUrl`, and use the `{ ... }` object to configure the method, headers, and body.

### Core rule table

| Module | Purpose |
|---|---|
| `Env` | Defines environment variables, such as API key, model name, and allowed origin |
| `RequestRecord` | Records request count for an IP, used for rate limiting |
| `requestWindowMs = 60_000` | One request window is 60 seconds |
| `maxRequestsPerWindow = 20` | Each IP can make at most 20 requests per minute |
| `maxBodyBytes = 12_000` | Request body limit is 12KB |
| `json(...)` | Quickly returns a JSON response |
| `corsHeaders(...)` | Sets CORS headers so the frontend can call this API |
| `isRateLimited(...)` | Checks whether the current IP is requesting too frequently |
| `extractJsonObject(...)` | Pulls `{...}` out of the model text and runs `JSON.parse` |
| `fetch(...)` | Worker entry point; all requests come through here |

So the frontend first uses:

```ts
buildAstrologyChartSeed(input);
```

to generate `chart`, then sends it to this API.

If `chart` is missing, it returns:

```json
{ "error": "missing_chart" }
```
