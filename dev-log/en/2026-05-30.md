# 2026-05-30 Development Log

Today I started by reading `brainTypes.ts`.

## `brainTypes.ts`

This file mainly defines two things:

1. What input does the agent brain receive?
2. What output must the agent brain return?

Core types:

```text
AgentBrainInput        = context shown to the brain
AgentBrainOutput       = reaction result returned by the brain
agentBrainOutputSchema = Zod schema used to validate the returned result
```

This file connects several systems:

```text
Initial Hand
    ↓
Agent current state
    ↓
Echo event
    ↓
Agent brain input/output
```

### `agentReactionSchema`

`agentReactionSchema` defines what reactions the agent can have.

When the player sends an Echo, the agent does not have to accept it directly. She can respond in 6 ways:

| reaction | meaning |
|---|---|
| `accepted` | received it |
| `hesitated` | hesitated |
| `resisted` | resisted it |
| `misread` | misread it |
| `delayed` | responded later |
| `transformed` | turned it into another understanding |

### `AgentBrainInput`

`AgentBrainInput` is the full context given to the brain.

```ts
export type AgentBrainInput = {
  language: "zh" | "en";
  profile: { ... };
  openingHand: { ... };
  currentState: AgentSignalState;
  relationships: Array<...>;
  dayContext: { ... };
  echoContext: { ... };
  memory: { ... };
  event: AgentBrainEvent;
};
```

This is the most important input type in the file. When the agent brain generates a reaction, it does not only look at a single Echo sentence.

It reads the profile, Initial Hand, current state, relationships, day context, Echo usage, recent memory, and current event.

### `agentBrainOutputSchema`

`agentBrainOutputSchema` defines what the brain must return.

This is the output validation shape:

```ts
export const agentBrainOutputSchema = z.object({
  behavior: z.object({ ... }),
  diary: z.object({ ... }),
  summary: z.object({ ... }),
  stateChanges: ...,
  relationshipChanges: ...,
  memory: z.object({ ... }),
  meta: z.object({ ... }),
});
```

This is very important. The final returned data must pass the schema.

### Data flow

The overall flow can be read like this:

```text
Player triggers Echo
    ↓
packaged as AgentBrainEvent

current profile / Initial Hand / current state / relationships / day / memory
    ↓
assembled into AgentBrainInput

fakeResolver or LLM resolver reads AgentBrainInput
    ↓
generates output

agentBrainOutputSchema.parse(output)
    ↓
trusted AgentBrainOutput
```

Then the output is distributed to:

- `behavior` → world-layer presentation
- `diary` → diary UI / day record
- `stateChanges` → runState / stateDrift
- `relationshipChanges` → relationshipDrift
- `memory.internalThought` → fed back into tomorrow's prompt

In one sentence: `brainTypes.ts` is the input/output protocol file for the agent brain layer.

## `echoResolution.ts`

The second file I read was `echoResolution.ts`.

It connects these steps:

```text
Player triggers Echo
    ↓
resolveBrain generates the agent's reaction
    ↓
stateDrift is calculated from the reaction
    ↓
relationshipDrift is calculated from the reaction
    ↓
everything is packed into a DailyEchoRecord
    ↓
complete EchoResolution is returned
```

This also shows an important boundary: the LLM or fakeResolver generates reaction and expression, but state changes, relationship changes, and daily records are still organized by local rules.
