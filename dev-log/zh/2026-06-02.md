# 2026-06-02 开发日记

今天主要从 `useWorldEcho.ts` 开始读，目的是把 `Echo` 这一层的实际闭环看清楚。

## `useWorldEcho.ts`

这个文件现在专门负责 `Echo` 相关流程：

```text
玩家打开信纸
  -> 写 Note Echo
  -> 发送给 LLM / fakeResolver
  -> 得到 agent 反应
  -> 计算状态漂移和关系漂移
  -> 更新 world state
```

### 核心数据流

```text
玩家写 Note Echo
  -> sendNoteEcho()
  -> buildNoteEchoInput()
  -> 组装 AgentBrainInput
  -> resolveAgentBrainLlm()
  -> 调用真实 LLM；失败时 fallback 到 fakeResolver
  -> resolveEchoOutcome()
  -> 计算 stateDrift + relationshipDrift + DailyEchoRecord
  -> dispatch({ type: "echo/applied", ... })
  -> worldReducer 更新世界状态
```

### `sendNoteEcho()`

`sendNoteEcho()` 是这个文件里最核心的函数。

它先读取玩家当前写下的草稿：

```ts
state.note.draft
```

然后用 `.trim()` 去掉前后空格。

如果玩家什么都没写：

```ts
if (!noteText) return;
```

就不会发请求。

### `echoApiUrl`

```ts
const echoApiUrl = import.meta.env.VITE_ECHO_API_URL;
```

这是 Vite 里读取环境变量的方式，意思是从 `.env` 里拿到 `VITE_ECHO_API_URL`，作为 `/echo` 请求的目标地址。

### 调用真实 LLM，并带 fallback

```ts
void resolveAgentBrainLlm({
  apiUrl: echoApiUrl,
  input,
  fallbackResolver: resolveAgentBrainFake,
}).then((brainOutput) => {
  ...
});
```

这里的意思是：

- 优先调用真实 LLM resolver
- 如果失败，就退回到 `resolveAgentBrainFake`

这里的 `void` 不是“没有结果”，而是明确表示：

- 我知道这里返回的是 Promise
- 但这里不 `await`
- 也不把这个 Promise 往外返回

### `resetForNextDay()`

```ts
const resetForNextDay = () => {
  dayStartAgentState.current = latestAgentState.current;
  dayStartRelationships.current = latestRelationships.current;
  latestDailyEchoes.current = [];
};
```

这个函数是在进入下一天时使用的。

它做三件事：

1. 把当天结束时的 `agentState` 作为新一天的开始状态
2. 把当天结束时的 `relationships` 作为新一天的开始关系
3. 清空 `dailyEchoes`

## 这一层最重要的理解

`useWorldEcho.ts` 最关键的地方其实就是 `sendNoteEcho()`：

```text
玩家写下一句话
  -> 拿到 noteText
  -> 组装成 AgentBrainInput
  -> 发给 LLM
  -> LLM 返回 brainOutput
  -> resolveEchoOutcome 做本地规则结算
  -> 更新 agentState / relationships / dailyEchoes
  -> dispatch 到 worldReducer
```

也就是说，这一层不是单纯“发一句话给模型”，而是把玩家信号接进本地世界规则。

## 一个 TypeScript 细节

`AgentBrainInput["event"]` 是 TypeScript 里的**索引访问类型**。

意思是：

- 从 `AgentBrainInput` 这个类型里
- 取出 `event` 字段对应的类型

它取的是**类型**，不是运行时的值。
