# 2026-05-28 开发日志

今天主要从 `WorldPrototype` 开始读文件。

## WorldPrototype

先看到这里：

```ts
export function WorldPrototype() {
  const { state, actions } = useWorldRuntime();
}
```

`useWorldRuntime` 是世界运行时的 hook，同时返回两个东西：`state` 和 `actions`。

`state` 会被拆给不同状态使用。

日记部分会判断 `diaryParagraphs` 上是否有真实内容：如果有，就显示真实日记；如果没有，就使用 fallback。

`traceSummary` 会把 echo 记录拼起来。

接下来是 `return` 的部分。`Application` 是 Pixi 的画布环境，`WorldStage` 才是真正画世界、跑世界画面的组件。

`WorldPrototype` 其实只是在说：

> 我给你一个画布，你在里面渲染世界。

比较理想的职责是：

```text
WorldPrototype
├── useWorldRuntime()       // 拿世界状态和 actions
├── Application             // Pixi 画布容器
│   └── WorldStage          // 世界舞台
├── NoteEchoDialog          // 纸条 UI
└── DiaryView               // 日记 UI
```

我现在的理解是：

```text
useWorldRuntime = 世界大脑
WorldPrototype = 接线板
WorldStage = 世界画面
NoteEchoDialog = 纸条交互层
DiaryView = 日记交互层
```

`WorldPrototype` 的核心数据流：

```text
useWorldRuntime
      ↓
state + actions
      ↓
WorldPrototype
      ↓
把 state 传给 WorldStage / Dialog / Diary
把 actions 传给 WorldStage / Dialog / Diary
      ↓
用户操作 UI
      ↓
调用 actions
      ↓
runtime 更新 state
      ↓
React 重新渲染
```

## WorldStage

下一个文件开始读 `WorldStage`。

首先说职责：它是真正放进 Pixi 画布里的“世界舞台”。

它负责把地图、纸条、NPC、Agent、Player 这些视觉层组合起来，并且通过 `useWorldLoop(props)` 启动世界循环逻辑。

```text
WorldStage
├── useWorldLoop(props)     // 世界循环、角色位置、碰撞、时间推进
└── pixiContainer           // Pixi 舞台容器
    ├── WorldMapSprite      // 地图
    ├── NotePaperSprite     // 纸条
    ├── NpcLayer            // NPC 层
    ├── AgentSprite         // Agent 角色
    └── PlayerSprite        // 玩家角色
```

```ts
extend({ Container, Graphics, Text });
```

这相当于注册 Pixi 组件。简而言之，就是启用 React 版 Pixi 语法里的 `Container / Graphics / Text`。

`WorldStageProps` 目前是：

```ts
type WorldStageProps = {
  day: number;
  paused: boolean;
  noteAvailable: boolean;
  agentState: AgentSignalState;
  echoEffect: EchoBehaviorEffect | null;
  onNotePicked: () => void;
  onDayComplete: () => void;
  onWorldContextChanged: (scene: WorldNodeId, timeOfDay: WorldTimeOfDay) => void;
};
```

核心调用是：

```ts
const { refs, snapshot } = useWorldLoop(props);
```

`WorldStage` 自己没有写移动逻辑、碰撞逻辑、时间推进逻辑，而是全部交给 `useWorldLoop(props)`。

它返回两个东西：

- `refs`：Pixi 对象的引用，用来直接操作画布上的对象。
- `snapshot`：当前这一帧，或者当前状态下的世界快照。

渲染顺序也很重要，大概是：

```text
最底层：地图
中间层：纸条
中间层：NPC
上层：Agent
最上层：Player
```

这里的层级关系可以理解成：

```text
WorldPrototype
负责 React 页面层：
- 创建 Pixi Application
- 管理弹窗
- 从 useWorldRuntime 拿 state/actions
- 把 props 传给 WorldStage

        ↓

WorldStage
负责 Pixi 舞台层：
- 注册 Pixi 元素
- 调用 useWorldLoop
- 渲染地图、纸条、NPC、Agent、Player

        ↓

useWorldLoop
负责世界循环：
- 玩家移动
- agent 移动
- 时间推进
- 捡纸条检测
- 一天结束检测
- 场景上下文变化
```

## useWorldLoop

接下来读 `useWorldLoop` 这个 hook。

文件职责：它是整个小世界里非常核心的游戏循环 hook。

当 `day` 变化时，会重置整个世界：

```ts
useEffect(() => {
  // reset world
}, [day]);
```

比如第一天结束，进入第二天，就需要：

- 玩家回到初始位置
- agent 回到初始路线
- 镜头重置
- 世界时间回到早晨
- 一天结束事件重新允许触发
- 学校停留状态清空
- 上下文记录清空

`useTick` 是真正的游戏循环。

这里是上下文变化上报：

```ts
if (lastContextKey.current !== worldContext.key) {
  lastContextKey.current = worldContext.key;
  onWorldContextChanged(worldContext.scene, worldContext.timeOfDay);
}
```

只有当上下文变化时，才会向 runtime 上报 context。

这个文件的主流程是：每一帧 `useTick` 执行：

1. 如果 `paused`，直接停止。
2. 计算 `dt`。
3. 根据键盘输入推进玩家和镜头。
4. 推进世界时间。
5. 根据玩家位置和时间生成世界上下文。
6. 如果上下文变化，通知 runtime。
7. 根据时间、agent 状态、echo 效果推进 agent。
8. 检测玩家是否按 `E` 捡纸条。
9. 检测一天是否结束。
10. 保存新的 `player / camera / agent` 状态。
11. 同步 Pixi 图形位置。
