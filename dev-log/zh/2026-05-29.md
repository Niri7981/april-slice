# 2026-05-29 开发日志

今天主要看了并整理了 Initial Hand 相关文件。

## 主要文件

### `initialHand.ts`

负责定义 `InitialHand` 的结构，并用 Zod 校验 interpreter 返回的数据。

这里的重点是：LLM 或 fallback 产出的结果不能直接相信，必须先经过 `initialHandSchema.parse(output)`，通过以后才是真正可信的 `InitialHand`。

### `astrologyInitialHand.ts`

一开始主要看的是这个方向：根据星盘/生日信息生成 Initial Hand。

后来这个思路进一步拆开：不应该由本地代码直接写死 tag，而应该让星盘信息进入解释流程，再输出结构化的 Initial Hand。

### `initialHandPrompt.ts`

负责把 `chart seed` 变成 prompt。

也就是说，它不直接生成 Initial Hand，而是把星盘资料、游戏 tone、输出规则和禁止项整理成给 interpreter 使用的提示词。

### `initialHandResolver.ts`

负责把几件事串起来：

- 调用 prompt builder。
- 调用 `interpret({ chart, prompt })`。
- 校验返回结果。

## Initial Hand 数据流

```text
AstrologyChartSeed
        ↓
buildInitialHandPrompt({ chart })
        ↓
prompt 字符串
        ↓
interpret({ chart, prompt })
        ↓
output / InitialHand 候选结果
        ↓
initialHandSchema.parse(output)
        ↓
真正可信的 InitialHand
```

## 今天的理解

Initial Hand 不应该是代码里提前写死的 personality 标签。

更合理的方向是：

```text
星盘资料
  -> LLM / interpreter 解读
  -> 结构化 Initial Hand tags
  -> Zod 校验
  -> 进入 run state 和 prompt
```

这些 tag 只作为开局倾向、语气、误读偏向和关系理解偏向使用，不直接决定行为、数值或剧情结果。
